# 🧠 PiPhone Agents

**Родина кишенькових AI-агентів для Android.**

Один агентний цикл — різні ролі. Спільне ядро, плагіни-гості, Cactus Needle всередині, Intents замість root.

---

## 🎯 Проблема

Ми хочемо **кишенькового Джарвіса**, який:
- Відправляє повідомлення за командою
- Прокладає маршрути
- Шукає та вмикає відео
- Відповідає на запитання
- Працює без інтернету (хоча б базово)
- Не жере батарею
- Не потребує root

Але на Android це розбивається об реальність:

| Перешкода | Деталі |
|---|---|
| **AccessibilityService** | Заблоковано на Xiaomi, Oppo, Huawei, Vivo |
| **`input tap` / `screencap`** | Тільки з root |
| **Termux** | Потужно, але не для звичайних користувачів |
| **GPS у фоні** | Батарея йде за години |
| **Великі LLM** | Потребують хмари, дорого, повільно |

Наше рішення не бореться з цими обмеженнями — воно їх **приймає і будує навколо них**.

---

## 💡 Наш підхід

### Три агенти, одне ядро

Замість одного монолітного «Джарвіса» ми робимо **трьох спеціалізованих агентів**:

| Агент | Роль | Тригер | Вихід |
|---|---|---|---|
| 🗺️ **Координатор** | Виконання команд | Текст / голос | Екран + TTS |
| 🎙️ **Секретар** | Hands-free асистент | Bluetooth-гарнітура | TTS |
| 🔔 **Проактивний** | Фоновий помічник | Час / подія / сповіщення | Notification |

Усі троє використовують **той самий агентний цикл** і **той самий набір плагінів**. Якщо ви написали плагін для WhatsApp — він працює в усіх трьох агентах одразу.

### Intents — це суперсила

Ми не намагаємося тапати кнопки в додатках. Ми використовуємо **Android Intents та URL schemes** — вони працюють на будь-якому телефоні, без root, без Accessibility:

```bash
# Telegram
tg://resolve?domain=username&text=привіт

# WhatsApp
https://wa.me/380XXXXXXXXX?text=привіт

# YouTube
vnd.youtube:VIDEO_ID

# Google Maps
geo:50.45,30.52?q=адреса
```

### Cactus Needle як reflex layer

[**Cactus Needle**](https://github.com/cactus-compute/needle) — це 26M параметрів, pure attention, без FFN-шарів. Він не вміє думати — він вміє **обирати інструмент**.

```
Запит → Cactus Needle (0.1с, 13MB)
         ├── > 75% → виконуємо (офлайн, безкоштовно)
         └── < 75% → LLM (хмара, 5-30с)
```

Це дає:
- Миттєву реакцію на прості команди
- Роботу без інтернету для базових дій
- Економію — хмара тільки для складного

### Жодного Accessibility

Xiaomi, Oppo, Huawei, Vivo — всі блокують AccessibilityService для сторонніх додатків. Ми не будуємо те, що не працюватиме на 80% телефонів.

**Не можемо:**
- Натиснути кнопку «Надіслати» у WhatsApp ❌
- Прочитати повідомлення з чату ❌
- Зробити скріншот ❌
- Ввести текст у поле ❌

**Можемо:**
- Відкрити WhatsApp з готовим повідомленням (тиснеш одну кнопку) ✅
- Відкрити YouTube з конкретним відео ✅
- Прокласти маршрут на Google Картах ✅
- Отримати GPS одноразово (без фонового режиму) ✅
- Відкрити Telegram чат з повідомленням ✅

### Зворотна сумісність

**Головне правило:** публічні API тільки додаються, ніколи не змінюються і не видаляються.

```
Плагін v1.0 + Ядро v1.0 → працює ✅
Плагін v2.0 + Ядро v1.0 → працює ✅ (новий плагін сумісний зі старим ядром)
Плагін v1.0 + Ядро v2.0 → працює ✅ (нове ядро сумісне зі старими плагінами)
```

Кожен плагін декларує версію API, з якою сумісний. Ядро не змінює публічні інтерфейси.

## 🧩 Як пишуться гості

Кожен «гість» — це Python-клас, який реєструє свої дії в глобальному ActionManager.

```python
from core.plugin_api import Plugin, Action

class WhatsAppGuest(Plugin):
    name = "whatsapp"
    version = "1.0.0"
    plugin_api_version = "1.0"
    description = "Взаємодія з WhatsApp через Intents"

    actions = {
        "send_message": Action(
            description="Відкрити WhatsApp з готовим повідомленням",
            params={"phone": str, "text": str},
            handler="whatsapp_send",
            examples=["напиши 380991234567 привіт"],
            requires_network=True,
            fallback_intent="https://wa.me/{phone}?text={text}",
        ),
    }

    def whatsapp_send(self, phone: str, text: str) -> dict:
        return {"intent": f"https://wa.me/{phone}?text={text}"}
```

Гість не залежить від агента — він працює з Координатором, Секретарем і Проактивним однаково.

## 🧠 Агентний цикл

Усі агенти використовують однаковий цикл:

```python
# plan → execute → check → (repeat or respond)

class AgentLoop:
    def plan(self, query):
        # Якщо Cactus Needle впевнений (> 75%) → використати його
        # Інакше → запитати LLM (OpenRouter, DeepSeek, Claude)
        
    def execute(self, plan):
        # Виконати дію: Intent, shell, API, або відповісти текстом
        
    def check(self, result):
        # Перевірити чи достатньо, або треба ще крок
```

Цей цикл не змінюється між агентами. Змінюється тільки:
- **Як отримується запит** (текст, голос, сповіщення)
- **Як виконуються дії** (Intent, shell, API)
- **Як дається відповідь** (текст, TTS, сповіщення)

## 🗺️ Що вміють гості (і що ні)

### Telegram
**✅** `tg://resolve?domain=username&text=...` — відкрити чат з повідомленням
**❌** Прочитати чат, натиснути «Надіслати»

### WhatsApp
**✅** `wa.me/phone?text=...` — відкрити з повідомленням
**❌** Пошукати контакт «Коля», натиснути кнопки

### YouTube
**✅** `vnd.youtube:VIDEO_ID` — відкрити та програти відео
**❌** Шукати за назвою (потрібен API), скачати, керувати плеєром

### Google Карти
**✅** `geo:lat,lon?q=query` — маршрути, пошук, навігація
**✅** GPS одноразово (без фону, без батареї)

**✅** Ліхтарик, Wi-Fi налаштування, батарея, Bluetooth
**❌** Скріншоти, блокування, root-команди

**✅** Google Keep через Intent (текст у буфері обміну)

## 🙋 Чому не «як Siri / Google Assistant / Alexa»?

Тому що вони:

| Асистент | Обмеження |
|---|---|
| **Siri** | Тільки Apple, закрита екосистема |
| **Google Assistant** | Шпигує, не можна кастомізувати |
| **Alexa** | Тільки Amazon, хмара |
| **Наш** | Будь-який Android, open source, Cactus Needle офлайн, Intents |

Siri не відправить повідомлення через Telegram. Google Assistant не відкриє конкретне відео на YouTube за назвою без голосового пошуку. Alexa не прокладе маршрут на Google Картах.

**Наш агент вміє все це — і ви можете дописати що завгодно.**
